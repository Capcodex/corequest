# Lexique produit — CoreQuest

## Objectif

Stabiliser le langage produit pour que CoreQuest paraisse plus crédible, plus cohérent et moins "MVP gamifié".

---

## Termes de référence

- `parcours` : l’ensemble du chemin d’apprentissage Rust dans le produit
- `niveau` : une unité de progression composée d’un cours court, d’un exercice et d’une validation
- `progression` : l’état global de l’utilisateur dans le parcours
- `avancement` : formulation alternative utile dans les résumés et libellés UI
- `XP` : la mesure chiffrée de progression
- `crabe` : l’avatar de progression ; un repère visuel, pas un narrateur
- `tableau de bord` : la vue de synthèse utilisateur
- `carte de progression` : la vue visuelle du parcours

---

## Termes à réduire ou éviter

- `compagnon` quand il devient le sujet principal du message
- `mission` hors contexte pédagogique précis
- `sécuriser` pour parler d’un exercice terminé
- `point de passage` pour parler d’un niveau suivant
- `point d’ancrage` pour décrire le tableau de bord
- `roadmap` quand `parcours` ou `carte de progression` suffit

---

## Règles de ton

- privilégier des phrases courtes et informatives ;
- éviter les métaphores inutiles dans les écrans cœur ;
- garder la gamification dans le système, pas dans chaque phrase ;
- faire apparaître clairement : état, bénéfice, action suivante.
